# PRD: Webcam AI CCTV

## 1. 제품명

**Webcam AI CCTV: 방치물·도난 의심 감지 시스템**

## 2. 제품 정의

웹캠 영상에서 사람과 물건을 실시간으로 탐지하고 추적하여, 사람이 물건을 두고 떠나는 상황과 다른 사람이 물건에 접근해 가져가는 상황을 **방치물** 및 **도난 의심** 이벤트로 판단하는 컴퓨터비전 시스템이다.

## 3. 배경

일반적인 객체인식 프로젝트는 화면 속 객체 이름을 표시하는 데서 끝난다. 본 프로젝트는 다음 단계를 추가한다.

```text
객체 탐지 → 객체 추적 → 관계 추론 → 시간 기반 이벤트 판정
```

이 구조는 단순 모델 데모가 아니라 실제 CCTV 분석 시스템에 가까운 경험을 제공한다.

## 4. 목표

### 제품 목표

- 웹캠 1대로 사람과 물건을 실시간 감지한다.
- 사람과 물건에 추적 ID를 부여한다.
- 사람-물건 근접 관계를 시간 기반으로 분석한다.
- 방치물 이벤트를 감지한다.
- 도난 의심 이벤트를 감지한다.
- 이벤트 로그와 캡처를 남긴다.
- Streamlit 대시보드로 이벤트를 확인한다.
- 커스텀 YOLO 학습으로 발표 환경 정확도를 개선한다.
- MacBook에서 최종 시연한다.

### 학습 목표

- 객체탐지 모델 활용
- 자동 라벨링 파이프라인 구축
- YOLO 커스텀 학습
- 객체 추적
- 이벤트 로직 설계
- 실시간 영상 처리
- 모델 평가와 런타임 벤치마크
- Codex 기반 바이브코딩 워크플로우 경험

## 6. 주요 사용 시나리오

### 시나리오 1: 방치물 감지

1. 사람이 가방을 들고 화면에 들어온다.
2. 가방을 책상이나 바닥에 내려놓는다.
3. 사람이 가방 근처에 3초 이상 머문다.
4. 시스템이 사람을 가방의 owner 후보로 등록한다.
5. 사람이 화면 밖으로 나간다.
6. 가방만 화면에 남아 있다.
7. 5초 후 시스템이 `abandoned_object` 이벤트를 발생시킨다.

### 시나리오 2: 도난 의심 감지

1. 가방이 방치물 상태다.
2. owner가 아닌 사람이 가방 근처에 접근한다.
3. 해당 사람이 가방을 들고 화면 밖으로 나간다.
4. 가방이 일정 시간 이상 탐지되지 않는다.
5. 시스템이 `theft_suspected` 이벤트를 발생시킨다.

### 시나리오 3: 정상 상황

1. 사람이 가방 근처에 머문다.
2. 가방이 계속 같은 위치에 있다.
3. owner도 계속 화면에 있다.
4. 시스템은 방치물로 판단하지 않는다.

## 7. 기능 요구사항

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-001 | 웹캠 입력을 지원한다. | Must |
| FR-002 | 영상 파일 입력을 지원한다. | Must |
| FR-003 | 사람/가방/노트북/휴대폰을 탐지한다. | Must |
| FR-004 | 객체 ID를 추적한다. | Must |
| FR-005 | 사람-물건 근접 시간을 계산한다. | Must |
| FR-006 | owner 후보를 등록한다. | Must |
| FR-007 | 방치물 이벤트를 감지한다. | Must |
| FR-008 | 도난 의심 이벤트를 감지한다. | Must |
| FR-009 | 이벤트 로그를 CSV로 저장한다. | Must |
| FR-010 | 이벤트 순간 캡처를 저장한다. | Must |
| FR-011 | OpenCV 화면에 박스/ID/상태를 표시한다. | Must |
| FR-012 | Streamlit 대시보드를 제공한다. | Should |
| FR-013 | 자동 라벨링 파이프라인을 제공한다. | Should |
| FR-014 | YOLO 학습/검증 스크립트를 제공한다. | Should |
| FR-015 | MacBook FPS 벤치마크를 제공한다. | Should |
| FR-016 | sample video fallback을 제공한다. | Must |

## 8. 비기능 요구사항

| ID | 요구사항 | 기준 |
|---|---|---|
| NFR-001 | MacBook에서 시연 가능 | FPS 10 이상 목표 |
| NFR-002 | Windows에서 학습 가능 | RTX 4070 SUPER CUDA 사용 |
| NFR-003 | 테스트는 GPU 없이 가능 | pytest 통과 |
| NFR-004 | 경로는 크로스 플랫폼 | pathlib 사용 |
| NFR-005 | 핵심 이벤트 로직은 모델 없이 테스트 가능 | mock sequence test |
| NFR-006 | 개인정보 식별 금지 | 얼굴인식/신원식별 없음 |
| NFR-007 | 도난 확정 표현 금지 | theft_suspected만 사용 |
| NFR-008 | 인터넷 없어도 시연 가능 | local model + sample video |
